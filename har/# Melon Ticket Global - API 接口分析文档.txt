# Melon Ticket Global - API 接口分析文档

> 分析时间: 2026-03-12
> 数据来源: 抓包 HAR 文件 (排队取号 / 购买票 / 取消票 完整流程)
> 目标平台: tkglobal.melon.com (PC端国际版)
> 运营方: Kakao Entertainment

---

## 一、平台概况

### 1.1 域名结构

| 域名 | 用途 |
|------|------|
| `tkglobal.melon.com` | PC端国际版主站（页面渲染） |
| `m.globalticket.melon.com` | 移动端国际版（PC访问会403或重定向） |
| `gmember.melon.com` | 用户认证/登录系统 |
| `cdnticket.melon.co.kr` | 静态资源CDN（座位图SVG等） |
| `dkpg-web.payments.kakao.com` | 支付网关（Kakao Pay） |
| `ticket.melon.com` | 韩版主站（非本项目目标） |

### 1.2 关键固定参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `pocCode` | `SC0002` | 销售渠道代码（国际版固定值） |
| `pocId` | `WP19` | 页面渠道标识 |
| `langCd` | `EN` | 语言代码 |
| `perfTypeCode` | `GN0001` | 演出类型（一般演出） |
| `sellTypeCode` | `ST0001` | 销售类型（一般销售） |
| `prodTypeCode` | `PT0001` | 产品类型 |

### 1.3 注册限制

仅支持以下邮箱域名注册: gmail.com, qq.com, yahoo.co.jp, icloud.com, outlook.com, hotmail.com

## 二、完整购票流程 (7个阶段)

```
登录 → 选择演出/场次 → 排队取号 → 选区域/座位 → 验证码 → 确认订单 → 支付
```

## 三、API 接口详解

### 阶段1: 用户认证

#### 1.1 登录
```
POST https://gmember.melon.com/login/login_proc.htm
Content-Type: application/x-www-form-urlencoded

请求参数:
  rtnUrl      = https://tkglobal.melon.com/main/index.htm  (登录后重定向URL)
  langCd      = EN
  email       = 用户邮箱
  pwd         = 用户密码

响应: HTTP 302 重定向到 rtnUrl，Set-Cookie 中包含会话信息
```

**重要**: 登录成功后浏览器会获得 Cookie，后续所有请求都需要携带这些 Cookie。

#### 1.2 检查登录状态
```
POST https://tkglobal.melon.com/main/ajax/isLogin.json

响应: {"result": 0}
  result=0 表示已登录
```

#### 1.3 获取会员Key
```
POST https://tkglobal.melon.com/member/getMemberKey.json

响应: {"memberKey": 1000570288}
```
memberKey 在后续的购票流程中会用到。

### 阶段2: 演出信息查询

#### 2.1 查询演出日期列表
```
GET https://tkglobal.melon.com/tktapi/glb/product/schedule/daylist.json

参数:
  callback    = scheduleList2          (JSONP回调名)
  prodId      = 212838                 (演出ID，从URL获取)
  pocCode     = SC0002
  perfTypeCode = GN0001
  sellTypeCode = ST0001
  langCd      = EN
  prodTypeCode = PT0001
  v           = 1
  timestamp   = 20260311200003         (格式: yyyyMMddHHmmss)

响应 (JSONP格式):
  perfDaylist: [
    {
      prodId: "212838",
      perfDay: "20260501",             // 演出日期
      rsrvStartDt: "20260311200000",   // 开售时间
      rsrvEndDt: "20260430165959",     // 停售时间
      groupSch: "100003"               // 场次分组号
    },
    ...
  ]
```

**捡漏关键**: 开售前 perfDaylist 为空数组，开售后才有数据。

#### 2.2 查询场次时间列表
```
GET https://tkglobal.melon.com/tktapi/glb/product/schedule/timelist.json

参数:
  callback    = scheduleList3
  prodId      = 212838
  perfDay     = 20260501               // 从 daylist 获取的日期
  pocCode     = SC0002
  perfTypeCode = GN0001
  sellTypeCode = ST0001
  seatCntDisplayYn = N
  langCd      = EN
  v           = 1

响应 (JSONP格式):
  perfTimelist: [
    {
      scheduleNo: "100003",            // ★ 场次编号，后续所有流程都需要
      perfDay: "20260501",
      perfTime: "1900",                // 演出时间 19:00
      cancelCloseDt: "20260430165959", // 取消截止时间
      sellCondNo: "1",
      rsrvStartDt: "20260311200000",
      rsrvEndDt: "20260430165959",
      seatPoc: "1"
    }
  ]
```

#### 2.3 查询座位等级列表
```
GET https://tkglobal.melon.com/tktapi/glb/product/schedule/gradelist.json

参数:
  callback    = scheduleList4
  prodId      = 212838
  pocCode     = SC0002
  scheduleNo  = 100003                 // 从 timelist 获取
  perfTypeCode = GN0001
  sellTypeCodeData = ST0001
  langCd      = EN
  seatCntDisplayYn = N
  v           = 1

响应:
  seatGradelist: [
    {
      seatGradeNo: "10009",            // 座位等级编号
      seatGradeName: "전석",            // 座位等级名称
      basePrice: "110000",             // 基础价格 (韩元)
      seatCount: "10000",              // 总座位数
      realSeatCntlk: "10000",          // ★ 实际可用座位数 (捡漏核心监控值)
      rsrvStartDt: "20260311200000",
      rsrvEndDt: "20260430165959"
    }
  ]
```

**★ 捡漏核心**: `realSeatCntlk` > 0 表示有可用票。此接口是监控的主要目标。

### 阶段3: 排队系统 (流量控制)

#### 3.1 获取排队Key
```
GET https://tkglobal.melon.com/tktapi/glb/product/prodKey.json

参数:
  callback    = scheduleList8
  prodId      = 212838
  scheduleNo  = 100003
  v           = 1

响应:
  {
    code: "0000",
    trafficCtrlYn: "Y",                // Y=需要排队, N=不需要
    key: "2X1bWVXKGnqLyh4cnWoH...",   // ★ 排队密钥 (加密)
    nflActId: "hBAQuLvqjosSkVIMMRQw"   // NetFunnel 活动ID
  }
```

**关键发现**: 当 `trafficCtrlYn = "Y"` 时，系统使用 **NetFunnel** 虚拟排队系统。
排队密钥 `key` 会在后续进入预订页面时作为 `chk` 参数传递。
`nflActId` 用于 NetFunnel 的排队流程。

#### 3.2 进入预订弹窗 (带排队Key)
```
POST https://tkglobal.melon.com/reservation/popup/onestop.htm

参数 (form-urlencoded):
  prodId       = 212771
  pocCode      = SC0002
  scheduleNo   = 100001
  sellCondNo   = 
  sellTypeCode = ST0001
  t            =
  tYn          = N
  chk          = fsqbko9mJiZA2Fd2bo6d...  // ★ 排队获得的加密Key
  langCd       = EN
  netfunnel_key =                          // NetFunnel排队完成后的key

响应: HTML页面 (预订弹窗的主框架)
```

## 四、捡漏程序核心策略

### 4.1 监控目标接口

**推荐方案**: 使用 `block/summary.json` 接口监控各区块余票

```
POST /tktapi/product/block/summary.json
参数: prodId, pocCode, scheduleNo

监控逻辑:
  for block in response.summary:
      if block.realSeatCntlk > 0:
          # 发现可用票！立即进入下单流程
          trigger_purchase(block)
```

**备选方案**: 使用 `gradelist.json` 接口按座位等级监控

```
GET /tktapi/glb/product/schedule/gradelist.json
参数: prodId, pocCode, scheduleNo

监控逻辑:
  for grade in response.seatGradelist:
      if int(grade.realSeatCntlk) > 0:
          trigger_purchase(grade)
```

### 4.2 JSONP 响应解析

大部分 API 返回 JSONP 格式，需要剥离回调函数名:

```python
import re, json

def parse_jsonp(text):
    """将 JSONP 响应解析为 dict"""
    match = re.search(r'\((.+)\);?\s*$', text, re.DOTALL)
    if match:
        return json.loads(match.group(1))
    return json.loads(text)  # 非JSONP直接解析
```

### 4.3 关键难点

| 难点 | 说明 | 可能的解决方案 |
|------|------|----------------|
| 验证码 (CAPTCHA) | 选座前需要输入图形验证码 | OCR识别 / 第三方打码平台 / 人工介入 |
| 排队系统 (NetFunnel) | 热门演出需要排队获取 key | 分析 NetFunnel JS 逻辑，模拟排队 |
| 加密参数 (chk, encryptedSeatIds) | 多处使用加密token | 需要分析前端JS生成逻辑 |
| 会话维持 | Cookie 过期需重新登录 | 定期刷新会话 / 检测登录状态 |
| 支付自动化 | 支付涉及银行卡认证跳转 | 仅自动化到下单步骤，支付手动完成 |
| 请求频率限制 | 过于频繁可能被封IP | 控制请求间隔，使用代理池 |

### 4.4 实际数据示例

**演出ID**: 212838 (排队HAR) / 212771 (购票HAR - QWER演唱会)
**场次号**: 100001 / 100003
**会员Key**: 1000570288 / 1003330177
